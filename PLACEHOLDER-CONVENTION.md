**AI West Africa (AIWA) — School Pilot Framework**
Version 1.0 | Effective Date: 1 May 2025

---

## 1. Purpose

This document defines the standard placeholder values used across all AIWA templates.
Consistent use of these placeholders ensures that templates remain clean, readable, and
easy to complete accurately.

All placeholders use square brackets and uppercase text: `[PLACEHOLDER_NAME]`

When completing a template, every placeholder must be replaced with actual information.
A completed document must contain **no remaining placeholder text**.

## 9. Completing a Template — Checklist

Before a document is considered complete and ready for use or execution:

- [ ] All `[PLACEHOLDER]` values have been replaced with actual information
- [ ] No placeholder text remains anywhere in the document, including headers and footers
- [ ] Dates are formatted according to the [Document Standards](DOCUMENT-STANDARDS.md)
- [ ] The document header (version, status, approval) has been updated
- [ ] The document has been reviewed by the appropriate authority
- [ ] The completed document has been saved with the correct executed file name
- [ ] A copy has been provided to all signing parties
